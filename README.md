# movie-multi-language-query

软件工程大作业：中外影视演员与作品多语言查询平台

## 项目构建内容

### 页面布局构建
搭建网页整体结构，划分头部标题模块、搜索筛选模块、查询结果模块与底部备注模块。完成文本输入框与地区、影片类型、上映年代、演员国籍四类下拉筛选控件的页面布局。

### 界面样式构建
单独创建样式文件，统一页面整体风格，对容器增加圆角与阴影效果，优化卡片排版，让网页界面整洁、视觉舒适，保证布局自适应。

### 查询功能构建
编写前端交互脚本，内置中外影视作品测试数据。实现中英文片名、演员姓名的模糊检索，支持多个筛选条件同时生效，实时过滤数据并动态渲染影片卡片，自动统计并展示结果条数。

### 项目工程化拆分
将页面结构、样式、业务脚本拆分为三个独立文件，代码职责分明，结构规范，方便后续接入后端接口，替换本地模拟数据。

## 互联网搜索体系接入
本项目已扩展后端服务，支持与互联网影视数据接口对接。通过配置 TMDB 或 OMDB API Key，前端查询请求会发送到本地后端，再由后端调用外部影视数据库进行搜索。

## 运行方式

1. 安装依赖：

```bash
npm install
```

2. 创建 `.env` 文件并填写 API Key：

```text
TMDB_API_KEY=你的_tmdb_api_key
OMDB_API_KEY=你的_omdb_api_key
```

3. 启动服务：

```bash
npm start
```

4. 打开浏览器访问：

```
http://localhost:3000
```

如果未配置外部 API Key，系统仍保留本地示例数据以保障演示功能。但只有配置 `TMDB_API_KEY` 才能检索到更为海量的互联网影视信息。

## 说明

- 前端页面通过 `script.js` 向 `/api/search` 发起请求。
- 后端在 `server.js` 中接收搜索参数，并优先查询本地数据与外部接口结果。
- TMDB 搜索已经覆盖电影、电视剧与演员演职员查询，返回更多真实互联网影视数据。
- 页面结果卡片已升级为海报图与信息并列展示，视觉更丰富。
- 若未成功返回外部结果，系统会自动回退到本地演示数据，保证页面继续可用。
